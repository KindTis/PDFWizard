# PDFWizard WASM 중심 설계서 (GitHub Pages)

- 작성일: 2026-04-19
- 범위: MVP 설계(구현 제외)
- 목표 배포: GitHub Pages (정적 호스팅)

## 1. 확정 요구사항

1. 서비스 형태
- GitHub Pages 기반 웹앱
- PDF 편집 기능 제공

2. 핵심 기능
- PDF 합치기
- PDF 분할
- 이미지 추출(원본 이미지 객체 추출)
- 페이지 -> 이미지 변환

3. 정책 및 제약
- 완전 로컬 처리(업로드 파일 외부 전송 금지)
- 모바일 포함 최신 주요 브라우저 지원
- 처리 규모 목표: 총 300MB, 2,000페이지
- 성능 목표: 100MB 기준 주요 작업 20초 내

4. UX 방향
- 메인 레이아웃: 작업흐름형(A)
- 분할 기능: 범위 입력 중심 + 페이지 썸네일 GUI 지원
- 출력 포맷: PNG/JPG 선택, 다중 결과 ZIP 다운로드

5. WASM 전략
- WASM 중심 아키텍처
- 기본 모드: 단일 스레드 WASM + Web Worker
- 확장 모드: 환경 충족 시 멀티스레드 가속

## 2. 아키텍처 설계

1. UI Shell
- 업로드, 작업 설정, 진행률, 결과 다운로드 화면

2. Job Orchestrator
- 작업 큐 관리
- 청크 분할/실행 순서 제어
- 취소/재시도/중단 복구 관리

3. WASM Worker Runtime
- PDF 연산 실행 전담
- 메인 스레드 블로킹 방지

4. WASM Engine Adapter
- 앱 내부 공통 인터페이스 제공
- 엔진 교체/확장 영향 최소화

5. Export Layer
- Blob 생성
- 다중 결과 ZIP 패키징
- 파일 다운로드 처리

## 3. 컴포넌트/화면 설계

1. Step Header
- 업로드 -> 작업 -> 내보내기 단계 표시

2. Upload Zone
- 드래그앤드롭/파일 선택
- 파일 목록/순서 관리
- 손상/암호/용량 경고

3. Thumbnail Workspace
- 페이지 썸네일 가상 스크롤
- 단일/다중/범위 선택
- 분할 GUI 연동

4. Action Panel
- 작업 탭: 합치기/분할/이미지 추출/페이지->이미지
- 작업별 옵션 입력
- 실행/취소

5. Job Progress & Result
- 진행률, 상태, 남은 시간
- 결과 미리보기/개별 다운로드/ZIP 다운로드

6. Mobile Adaptation
- 1열 우선 레이아웃
- 하단 시트 및 전체화면 전환
- 저사양 자동 품질 하향

## 4. 데이터 흐름

1. 입력
- PDF 업로드 후 File Registry에 메타 저장
- 원본 데이터는 로컬 메모리/임시 저장소 유지

2. 인덱싱
- Worker에서 페이지 구조 분석
- 썸네일 생성 작업 큐 등록

3. 작업 설정
- Job Spec 생성(타입/범위/포맷/품질)
- 버전 필드로 동시성 충돌 방지

4. 실행
- Job Orchestrator가 청크 단위 실행
- 진행 이벤트 및 부분 결과 스트리밍
- 취소 요청 시 안전 중단

5. 출력
- 단일 결과 Blob 다운로드
- 다중 결과 ZIP 묶음 다운로드
- 실패 항목 별도 리포트

6. 복구
- 메모리 임계 시 청크 축소
- 실패 청크 재시도
- 부분 성공/부분 실패 분리 표시

## 5. 에러 처리 정책

1. 에러 분류
- 입력/리소스/연산/출력/환경 에러

2. 사용자 메시지 원칙
- 원인 + 즉시 가능한 조치 제시
- 치명 오류 vs 부분 실패 명확 분리

3. 자동 복구
- 청크 축소 후 재시도
- 썸네일 품질 하향
- Worker 재기동 1회 시도

4. 상태 모델
- idle -> validating -> running -> paused -> completed | partial_failed | failed | cancelled

5. 로컬 로그
- 사용자 선택 시 JSON 내보내기
- 민감 데이터/원문 저장 금지

6. 호환성 폴백
- 멀티스레드 불가 시 단일 스레드 자동 전환
- 미지원 기능 단계적 비활성화

## 6. 테스트 전략

1. 단위 테스트
- 범위 파서, 상태 전이, 파일명 규칙, 메시지 매핑

2. 통합 테스트
- Worker + WASM + 오케스트레이터 흐름
- 취소/재시도/부분 실패

3. E2E 테스트
- 업로드 -> 썸네일 선택 -> 실행 -> 다운로드
- 모바일 터치 시나리오 포함

4. 성능 테스트
- 100MB 기준 20초 목표 검증
- 병목 구간 프로파일링

5. 호환성 테스트
- Chrome/Edge/Firefox/Safari 최신 주요 버전
- 데스크톱/모바일 매트릭스 운영

6. 안정성 테스트
- 메모리 압박/Worker 종료/복구 시나리오

7. 보안/프라이버시 테스트
- 네트워크 전송 없음 확인
- 로그 민감정보 미기록 확인

## 7. 결정 로그

1. 호스팅/실행
- GitHub Pages + 클라이언트 전용 처리

2. 처리 기준
- 총 300MB / 2,000페이지

3. 보안
- 완전 로컬 처리

4. UX
- 작업흐름형 레이아웃(A)
- 분할은 범위 입력 + 썸네일 GUI

5. 이미지 출력
- PNG/JPG 선택 + ZIP

6. WASM 운영
- WASM 중심, 기본 단일 스레드, 조건부 멀티스레드

## 8. 구현 전 준비 항목(설계 범위)

1. 후보 WASM 엔진 비교표 작성(기능/번들/라이선스)
2. 샘플 PDF 코퍼스 정의(정상/손상/암호/대용량)
3. 브라우저 호환성 매트릭스 초안 작성
